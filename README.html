OneTaste Family 一家一味 - 项目文档

项目概述

项目名称

OneTaste Family（一家一味）

项目背景

现代生活中，很多人面临选择困难症，每天不知道想吃什么、吃什么适合自己、对自己身体好。对于家庭主妇来说，买菜也是一个难题，需要买什么菜、买多少、可以存放多久等问题也困扰着大家。

项目目的

解决家庭日常生活中的核心问题：

  • 吃什么：帮助用户决定每天想吃什么
  • 应该吃什么：根据身体状况推荐合适的食物
  • 买什么：根据菜单生成购物清单
  • 买多少：合理规划食材采购数量

项目定位

面向家庭的智能食谱管理和购物助手系统，通过AI技术提供个性化的饮食建议和便捷的购物管理。

技术架构

架构模式

BS架构（Browser/Server）

技术栈

  • 后端：Go语言
  • 前端：Vue.js
  • AI服务：Python（大模型调用）
  • 数据库：PostgreSQL

系统组成

  1. 管理后台：用于系统管理和运营
  2. 用户小程序：面向家庭用户的移动端应用
  3. AI服务：独立的Python服务，提供大模型能力

功能需求

一、基础功能（免费版）

1. 家庭管理

  • 功能描述：用户组建成家庭
  • 详细需求：
    • 创建家庭
    • 邀请家庭成员加入
    • 家庭成员管理（添加、删除、角色设置）
    • 家庭信息管理

2. 家庭食谱管理

  • 功能描述：建立家庭食谱库
  • 详细需求：
    • 添加菜式（最多30个菜式）
    • 输入菜式名称
    • 输入对应食材（名称、数量、单位）
    • 输入烹饪步骤（支持文字和语音输入）
    • 编辑、删除菜式
    • 菜式分类管理
    • 支持语音输入转文字

3. 菜单规划

  • 功能描述：用户选择每天、每顿想吃什么，形成菜单
  • 详细需求：
    • 选择日期和餐次（早餐、午餐、晚餐）
    • 从家庭食谱中选择菜式
    • 生成每日菜单
    • 生成每周菜单
    • 菜单查看和编辑
    • 菜单分享给家庭成员

4. 购物清单生成

  • 功能描述：根据菜单对应的食材，生成购物清单
  • 详细需求：
    • 根据选择的菜单自动生成购物清单
    • 合并相同食材，计算总数量
    • 食材分类展示（蔬菜、肉类、调料等）
    • 购物清单编辑（增删改）
    • 购物清单标记（已购买、待购买）
    • 食材存放时间提醒

二、高阶功能（付费版）

0. 扩展食谱容量

  • 功能描述：增加家庭食谱数量上限
  • 详细需求：
    • 免费版：最多30个菜式
    • 付费版：最多60个菜式

1. AI语音菜单整理

  • 功能描述：语音输入菜单，AI大模型进行整理
  • 详细需求：
    • 支持语音输入菜式信息
    • AI去除语气词
    • AI修正逻辑不通、不顺的地方
    • 整理成规范的菜式格式
    • 限制：一天10次

2. 身体状况录入与分析

  • 功能描述：录入身体状况，AI分析用户身体状态
  • 详细需求：
    • 录入身体状况信息：
      • 疾病信息（得了什么病）
      • 工作状态（最近忙不忙、压力大不大）
      • 身体感觉（身体什么感觉）
    • AI分析用户身体状态
    • 生成健康建议
    • 限制：每天每人分析2次

3. AI智能菜单生成

  • 功能描述：AI根据用户身体状态分析生成菜单
  • 详细需求：
    • 基于用户身体状态
    • 结合家庭食谱库
    • 生成推荐菜单（每日/每周）
    • 说明推荐理由
    • 限制：一周生成5次

4. AI菜单合理性分析

  • 功能描述：AI分析菜单是否合理
  • 详细需求：
    • 分析菜单营养搭配
    • 分析菜单是否适合身体状况
    • 提供优化建议
    • 限制：一周5次

5. AI烹饪步骤整合

  • 功能描述：AI根据菜单里面各个菜式的烹饪步骤，整合成一顿饭的烹饪步骤
  • 详细需求：
    • 分析多道菜的烹饪步骤
    • 优化烹饪顺序
    • 整合成高效的烹饪流程
    • 提高做饭效率
    • 提供时间规划建议

数据库设计（概要）

核心表结构

用户表（users）

  • 用户ID、用户名、手机号、密码、创建时间等

家庭表（families）

  • 家庭ID、家庭名称、创建人、创建时间等

家庭成员表（family_members）

  • 成员ID、家庭ID、用户ID、角色、加入时间等

菜式表（dishes）

  • 菜式ID、家庭ID、菜式名称、分类、创建时间等

食材表（ingredients）

  • 食材ID、菜式ID、食材名称、数量、单位等

烹饪步骤表（cooking_steps）

  • 步骤ID、菜式ID、步骤序号、步骤内容等

菜单表（menus）

  • 菜单ID、家庭ID、日期、餐次、创建人、创建时间等

菜单菜式关联表（menu_dishes）

  • 关联ID、菜单ID、菜式ID等

购物清单表（shopping_lists）

  • 清单ID、家庭ID、生成日期、状态等

购物清单项表（shopping_list_items）

  • 项ID、清单ID、食材名称、数量、单位、状态等

身体状况记录表（health_records）

  • 记录ID、用户ID、记录内容、分析结果、记录时间等

会员表（memberships）

  • 会员ID、用户ID、家庭ID、会员类型、到期时间等

AI调用记录表（ai_usage_logs）

  • 记录ID、用户ID、调用类型、调用时间、剩余次数等

API设计（概要）

用户相关

  • POST /api/v1/auth/register - 用户注册
  • POST /api/v1/auth/login - 用户登录
  • GET /api/v1/user/info - 获取用户信息
  • PUT /api/v1/user/info - 更新用户信息

家庭相关

  • POST /api/v1/family/create - 创建家庭
  • GET /api/v1/family/info - 获取家庭信息
  • POST /api/v1/family/member/invite - 邀请成员
  • GET /api/v1/family/members - 获取家庭成员列表

食谱相关

  • POST /api/v1/dishes - 创建菜式
  • GET /api/v1/dishes - 获取菜式列表
  • GET /api/v1/dishes/{id} - 获取菜式详情
  • PUT /api/v1/dishes/{id} - 更新菜式
  • DELETE /api/v1/dishes/{id} - 删除菜式
  • POST /api/v1/dishes/voice-input - 语音输入菜式（付费）

菜单相关

  • POST /api/v1/menus - 创建菜单
  • GET /api/v1/menus/daily - 获取每日菜单
  • GET /api/v1/menus/weekly - 获取每周菜单
  • PUT /api/v1/menus/{id} - 更新菜单
  • POST /api/v1/menus/ai-generate - AI生成菜单（付费）
  • POST /api/v1/menus/ai-analyze - AI分析菜单（付费）

购物清单相关

  • POST /api/v1/shopping-lists/generate - 生成购物清单
  • GET /api/v1/shopping-lists/{id} - 获取购物清单
  • PUT /api/v1/shopping-lists/{id} - 更新购物清单
  • POST /api/v1/shopping-lists/{id}/items/{itemId}/toggle - 切换购物项状态

身体状况相关（付费）

  • POST /api/v1/health/record - 录入身体状况
  • POST /api/v1/health/analyze - AI分析身体状况
  • GET /api/v1/health/records - 获取身体状况记录

烹饪步骤相关（付费）

  • POST /api/v1/cooking/optimize - AI优化烹饪步骤

支付相关

  • POST /api/v1/payment/subscribe - 订阅会员
  • GET /api/v1/payment/membership - 获取会员信息

快速开始（Docker 部署）

数据库服务部署

使用 Docker 快速部署 PostgreSQL 和 Redis：

# 1. 进入 docker 目录
cd docker

# 2. 初始化环境
chmod +x init.sh
./init.sh

# 3. 编辑环境变量（设置强密码）
nano .env

# 4. 启动服务
docker-compose up -d

# 5. 验证服务
docker-compose ps
